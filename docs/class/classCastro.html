Advance grids at this level in time.

The main driver for a single level. This will do either the SDC algorithm or the Strang-split reactions algorithm.

  • dt: the timestep to advance (e.g., go from time to time + dt)

void apply_source_term_predictor()¶

Optionally predict the source terms to t + dt/2, which is the time-level n+1/2 value, To do this we use a lagged predictor estimate: \( \frac{dS}{dt_n} = (S_n - S_{n-1}) / dt \), so \( S_{n+1/2} = S_n + (dt / 2) \frac{dS}{dt_n} \). We’ll add the S_n terms later; now we add the second term. We defer the multiplication by dt / 2 to ::initialize_do_advance since for a retry we may not yet know at this point what the advance timestep is.

Note that since for dS/dt we want \( (S^{n+1} - S^{n}) / dt \), we only need to take twice the new-time source term, since in the predictor-corrector approach, the new-time source term is \( \frac{1}{2} S^{n+1} - \frac{1}{2} S^{n} \). This is untrue in general for the non-momentum sources, so for safety we’ll only apply it to the momentum sources.

Note that if the old data doesn’t exist yet (e.g. it is the first step of the simulation) FillPatch will just return the new data, so this is a valid operation and the result will be zero, so there is no source term prediction in the first step.

void swap_state_time_levels(const amrex::Real dt)¶

This is a hack to make sure that we only ever have new data for certain state types that only ever need new time data; by doing a swap now, we’ll guarantee that allocOldData() does nothing. We do this because we never need the old data, so we don’t want to allocate memory for it.

Parameters
  • dt: timestep

Real locWgtSumOneSide(const std::string &name, amrex::Real time, int idir, int side, int bdir, bool local = false)¶

This function is a clone of ::locWgtSum except that it only sums over one half of the domain. The lower half corresponds to side == 0, and the upper half corresponds to side == 1. The argument idir (x == 0, y == 1, z == 2) gives the direction to location weight by, and the argument bdir gives the direction along which to bisect.

Parameters
  • name: Name of quantity
  • time: current time

Real enforce_min_density(amrex::MultiFab &S_old, amrex::MultiFab &S_new, int ng)¶

This routine sets the density in S_new to be larger than the density floor. Note that it will operate everywhere on S_new, including ghost zones. S_old is present so that, after the hydro call, we know what the old density was so that we have a reference for comparison. If you are calling it elsewhere and there’s no meaningful reference state, just pass in the same MultiFab twice.

The return value is the the negative fractional change in the state that has the largest magnitude. If there is no reference state, this is meaningless.

Parameters
  • S_old: Old state
  • S_new: New state
  • ng: number of ghost cells
